Keep it simple!

The idea is to be simple! There are many PHP frameworks on the internet with generic solutions and dozens of options to perform simple tasks, using complex models and documentation tiresome. The MagicPHP has a different proposal, start your application with the minimum possible and use demand modules, has complete freedom to use any feature already available in a simple way.

On Demand

The concept of demand loading is already being used in languages ​​like Ruby and Node.js, now is the turn of PHP!, With the repository Magic is possible to develop smaller modules to solve simple problems and share with the community.

Hello World

See how simple it is to create your system in a few simple steps:

<?php
            /**
             * Hello World
             * 
             * @package     MagicPHP
             * @author      André Henrique da Rocha Ferreira <andrehrf@gmail.com>
             * @link        https://github.com/andrehrf/magicphp MagicPHP(tm)
             * @license     MIT License (http://www.opensource.org/licenses/mit-license.php)
             */

            Routes::SetDynamicRoute(function($sUri, $sMethod) {
                Output::SetNamespace("hello world");
                Output::SetTemplate(Storage::Join("dir.shell.default.tpl", "index.tpl"));
                Output::AppendCSS(Storage::Join("dir.shell.default.css", "index.css"));
                Output::Send();
            });

Modules

The structure of the modules is simple, everyone should be located in the /modules directory, the files needed for the operation are include.php and status.txt. File include.php will be responsible for adding information to the system module directory as __autoload is still the file that contains the module information such as author, license, website, among others. The status.txt file only exists to control the module activity, so as not to require control by the database, its content is 1 or 0 and 1 for when the module is active in the system, so you can keep the files modules and only use them when necessary.

Below an example of the include.php file:

<?php
            /**
             * Initializer module
             * 
             * @package     MagicPHP Development
             * @author      André Henrique da Rocha Ferreira <andrehrf@gmail.com>
             * @link        https://github.com/andrehrf/magicphp-development MagicPHP Development(tm)
             * @license     MIT License (http://www.opensource.org/licenses/mit-license.php) 
             */

            $oModule = Modules::Append("development", __DIR__ . SP);
            $oModule->Set("name", "MagicPHP Development")
                    ->Set("author", "André Ferreira <andrehrf@gmail.com>")
                    ->Set("website", "https://github.com/andrehrf/magicphp-development")
                    ->Set("licence", "MIT License")
                    ->Dependence("magicphp", "0.1")
                    ->Start();
